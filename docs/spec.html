Backlog Dashboard 仕様書Backlog Dashboard — Specification

Backlog の自分のタスクを扱う macOS メニューバー常駐アプリ。現状（v0.4.0）の実装に基づく仕様。

A macOS menu-bar resident app for your own Backlog tasks. Specification based on the current implementation (v0.4.0).

macOS · Apple SiliconElectron 32Backlog REST API v22026-07-23

01概要Overview

メニューバーの「b」アイコンをクリックするとポップアップが開き、未完了タスクの確認・更新、新規タスクの起票、Backlog 通知の受信・確認ができる。

  • 個人利用向けの単一ユーザーアプリ。各ユーザーは自分の Backlog API キーで接続する。
  • Backlog の REST API v2 を直接呼び出す（MCP や中間サーバーは介さない）。
  • Electron 製。UI はフレームワーク非依存の素の HTML/CSS/JS。

Clicking the "b" icon in the menu bar opens a popover where you can review and update open tasks, create new tasks, and receive/review Backlog notifications.

  • Single-user, personal-use app. Each user connects with their own Backlog API key.
  • Calls the Backlog REST API v2 directly (no MCP, no intermediary server).
  • Built with Electron. The UI is plain HTML/CSS/JS with no framework.

03画面仕様Screens

共通ヘッダー: ‹ 戻る / タイトル / 🔔 通知（未読バッジ） / + 新規 / ⟳ 更新 / ⚙ 設定。ウィンドウはフォーカス喪失・閉じるボタンで非表示になる（破棄されない）。 Common header: ‹ Back / title / 🔔 Notifications (unread badge) / + New / ⟳ Refresh / ⚙ Settings. The window hides (is not destroyed) on focus loss or close.

タスク一覧 — TasksTask list — Tasks

  • 自分が担当かつ未完了（statusId 1〜3）の課題。
  • 並び替え: 締切日順（既定・締切なしは末尾）／プロジェクト順／タスク名順。
  • 絞り込み: プロジェクト＋タスク名の部分一致（件名・課題キー）。
  • 締切バッジ（超過=赤／間近=橙）、右下に「表示件数 / 全件」。
  • Issues assigned to you and open (statusId 1–3).
  • Sort: due date (default; no-due last) / project / task name.
  • Filter: project + task-name substring (summary, issue key).
  • Due badge (overdue = red / soon = orange), "visible / total" bottom-right.

課題詳細 — IssueIssue detail — Issue

  • 件名・メタ情報、「Backlog で開く ↗」。
  • ステータス変更（プロジェクトのステータス一覧から）。
  • 説明・コメント一覧（変更ログ含む）。
  • コメント投稿。更新後は再取得して再描画。
  • Summary, meta, "Open in Backlog ↗".
  • Status change (from the project's status list).
  • Description and comment list (incl. change log).
  • Comment form. Refetches and re-renders after updates.

新規タスク — New TaskNew Task

  • プロジェクト（必須）→ 種別（必須）／件名（必須）。
  • 優先度（既定=中）／期限／説明。
  • プロジェクトは「最近使った順」— 起票実績を最新順（最大 5 件）で Recent グループの先頭に、残りは All projects に表示。
  • 作成成功でトースト表示 → 一覧へ戻る。
  • Project (required) → issue type (required) / summary (required).
  • Priority (default: Normal) / due date / description.
  • Project picker is "recently used first" — up to 5 most-recent under a Recent group, the rest under All projects.
  • Toast on success → returns to the list.

通知一覧 — NotificationsNotifications

  • 最新 100 件を新しい順。未読は緑でハイライト。
  • 差出人・種別・対象課題（Key-keyId）・日時。
  • 行クリック→課題詳細へ＋既読化。「Mark all read」で全既読。
  • 各行の Mark read（未読時のみ）でその通知だけ既読化＋バッジを 1 減。
  • 一覧を開いている間に新着を検知すると背景で自動更新。
  • Latest 100, newest first. Unread highlighted in green.
  • Sender, reason, target issue (Key-keyId), time.
  • Click → issue detail + mark read. "Mark all read" clears all.
  • Per-row Mark read (unread only) reads just that item + drops the badge by one.
  • An open list refreshes in the background when new activity is detected.

設定 — SettingsSettings

  • スペースドメイン（例: yourspace.backlog.com）と API キー。
  • 保存時に /users/myself で疎通確認。
  • 「Open API key settings ↗」で発行ページを開く。
  • Space domain (e.g. yourspace.backlog.com) and API key.
  • Verifies via /users/myself on save.
  • "Open API key settings ↗" opens the issuance page.

04通知（デスクトップ通知）Desktop notifications

  • メインプロセスが 60 秒間隔（POLL_INTERVAL_MS）で通知 API をポーリング。
  • 新着判定: 保存済み lastNotificationId より大きい ID かつ未読を新着とみなし macOS ネイティブ通知を表示。
  • 初回起動時は最大 ID を基準保存するだけで通知は出さない（起動時スパム防止）。
  • 通知タイトル=「差出人＋種別」、本文=「課題キー＋件名」。クリックで前面化し該当課題を表示。
  • 未読数をメニューバーアイコン横とヘッダーのベルバッジに反映（メイン側が unreadCount を保持）。
  • 既読時のバッジは楽観的更新: count API の反映遅延を避け、既読操作で即座に -1 → 次回ポーリングで整合。
  • 新着時は背景更新: notifications:new を送り、開いている一覧を背景で再描画。
  • lastNotificationId は notify-state.json に永続化（再起動後の再通知を防止）。
  • The main process polls the notifications API every 60s (POLL_INTERVAL_MS).
  • New-item detection: IDs greater than the stored lastNotificationId and still unread are shown as native macOS notifications.
  • On first launch, only the max ID is stored as a baseline — nothing fires (prevents startup spam).
  • Title = "sender + reason", body = "issue key + summary". Click brings the app forward and opens the issue.
  • The unread count shows next to the menu-bar icon and on the header bell badge (main holds unreadCount).
  • Optimistic badge on read: to dodge the count API's lag, a read decrements by one immediately; the next poll reconciles.
  • Background refresh on new activity: notifications:new re-renders an open list in the background.
  • lastNotificationId is persisted to notify-state.json (no re-notify after restart).

セキュリティ: contextIsolation: true / nodeIntegration: false。API キーはメインプロセスのみが保持し、レンダラーには渡さない。CSP で外部リソースを禁止。レンダラーに innerHTML 経路は無い（全てテキストノード）。

ウィンドウ挙動: dock 非表示のアクセサリアプリ（LSUIElement）。メニューバー直下・右寄りに配置（トレイ座標が不正な場合のフォールバック付き）。setVisibleOnAllWorkspaces + alwaysOnTop で現在の Space／全画面上に表示。終了はトレイ右クリックの Quit のみ。

Security: contextIsolation: true / nodeIntegration: false. The API key stays in the main process, never passed to the renderer. CSP forbids external resources. The renderer has no innerHTML path (everything is a text node).

Window behavior: dock-hidden accessory app (LSUIElement). Anchored just below the menu bar, toward the right (with a fallback for invalid tray coordinates). setVisibleOnAllWorkspaces + alwaysOnTop keep it above the current Space / full-screen apps. Truly quits only via the tray's Quit.

06データと永続化Data & persistence

  • 設定: ~/Library/Application Support/backlog-dashboard/config.json（{ spaceDomain, apiKeyEnc }、パーミッション 600）。
  • API キーは macOS Keychain で暗号化保存（Electron safeStorage）。ディスク上には暗号文（apiKeyEnc）のみ、復号はメモリ上のみ。旧版の平文キーは起動時に自動移行。
  • 通知状態: 同ディレクトリの notify-state.json（{ lastNotificationId }）。
  • 外部送信は Backlog API のみ。open:external は http(s) の URL のみを開く。
  • Config: ~/Library/Application Support/backlog-dashboard/config.json ({ spaceDomain, apiKeyEnc }, permission 600).
  • The API key is encrypted at rest via the macOS Keychain (Electron safeStorage). Only the ciphertext (apiKeyEnc) is on disk; decrypted only in memory. Legacy plaintext keys are auto-migrated on launch.
  • Notification state: notify-state.json in the same directory ({ lastNotificationId }).
  • The only outbound traffic is to the Backlog API. open:external opens only http(s) URLs.

09既知の制約 / 非対応Known limitations

  • Apple Silicon 専用（Intel Mac 非対応。ユニバーサルビルド未対応）。
  • 未署名のため配布時に Gatekeeper の手当てが必要。
  • 未完了タスクは statusId 1〜3 のみ対象（独自の完了系カスタムステータスは想定外）。
  • タスク・通知とも最大 100 件（ページングなし）。
  • 課題を伴わない通知（PR 等）はアプリ内詳細に遷移せず Backlog をブラウザで開く。
  • 自動起動は手動設定。Slack 連携は未実装。
  • Apple Silicon only (no Intel; universal build unsupported).
  • Unsigned, so distribution needs a Gatekeeper workaround.
  • Open tasks cover only statusId 1–3 (custom "closed-like" statuses are out of scope).
  • Tasks and notifications are capped at 100 items (no pagination).
  • Notifications without an issue (PRs, etc.) open Backlog in the browser, not an in-app detail.
  • Auto-start is manual. Slack integration is not implemented.
